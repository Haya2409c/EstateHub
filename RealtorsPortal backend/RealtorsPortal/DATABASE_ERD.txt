CARDINALITY SUMMARY
═══════════════════

One-to-Many (1:M):
  Country    → Region    (1:M, Cascade Delete)
  Region     → City      (1:M, Cascade Delete)
  City       → Area      (1:M, Cascade Delete)
  Area       → Property  (1:M, SetNull on delete)
  Category   → Property  (1:M, Restrict delete)
  Agent      → Property  (1:M, SetNull on delete)
  Property   → PropertyImage (1:M, Cascade delete)

One-to-One (1:1):
  None currently

Many-to-Many (M:M):
  None currently (future: Property ↔ Amenity)


KEY CONSTRAINTS
═══════════════

PRIMARY KEYS:
  All entities use IDENTITY INT PK
  Exception: Properties use surrogate key Int, but queryable by Slug

FOREIGN KEYS:
  Most use Restrict or Cascade delete behavior
  Agent FK uses SetNull (property can exist without agent)
  Area FK uses SetNull (property can exist without area)
  Geographic FKs use Cascade (when region deleted, cities/areas/properties deleted)

UNIQUE CONSTRAINTS (Index):
  Category.Slug - Non-unique, for SEO/routing
  News.Slug - Non-unique, for SEO/routing
  Property.Slug - Non-unique (future: unique per agent/area?)
  Agent.Slug - Non-unique

INDEXES (Performance):
  Property: 
    - (CategoryId, IsFeatured, ListedDate DESC)
    - (AreaId, Price, ListedDate DESC)
    - (AgentId, ListedDate DESC)
    - (Price, Bedrooms, Bathrooms)

  PropertyImage:
    - (PropertyId, IsPrimary DESC)

  Geographic:
    - (CountryId), (RegionId), (CityId) for JOINs

CONSTRAINTS & BUSINESS RULES
═════════════════════════════

1. Geographic Hierarchy:
   - A Property must exist in an Area
   - An Area must belong to a City
   - A City must belong to a Region
   - A Region must belong to a Country
   - Cascading deletes ensure referential integrity

2. Property Listing:
   - CategoryId is REQUIRED (what type of property)
   - AgentId is OPTIONAL (can be unassigned)
   - AreaId is OPTIONAL (can be generic/not located)
   - Must have at least 0 images (but typically 1+ primary)

3. Images:
   - IsPrimary = true for thumbnail on listing
   - SortOrder determines gallery order
   - Multiple IsPrimary = true allowed (app logic handles first)

4. Contact Messages:
   - IsRead flag for admin dashboard
   - SubmittedAt auto-set to UTC now

5. News:
   - PublishedAt auto-set to UTC now
   - No edit history (immutable after publish)

6. Agents:
   - Slug used for URL routing (/agents/profile/hifza)
   - Rating stored as double (0-5.0)
   - YearsExperience can be null
